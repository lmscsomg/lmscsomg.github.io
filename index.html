MIT-6.828_Lab-1

Mon 30 March 2015

因为学过微机和汇编，所以很多概念还是挺容易理解的。但纸上得来终觉浅，躬行之后还是受益良多的，比如对指针、对esp、ebp的概念理解。当然既然是操作系统课程，对很多操作系统底层的理解也更深入了。

PC Bootstrap

汇编使用的是AT&T标准的，和我们熟悉的Intel的那一套有一点区别，但是看手册理解起来很快（国外好像都喜欢用AT&T这一套的，比如CSAPP就用的是AT&T格式的）。

8086实模式，实际地址CS*16+IP。80286引入保护模式，分段机制。80386采用32位架构，段页式内存管理。为兼容性，RAM分为底部640KB以及0x00100000上面部分，而Lab只使用物理内存前256MB。

ROM BIOS

PC启动后，BIOS被执行，第一条指令CS:f000,IP:fff0,跳转到前面的地址执行指令......(有点蛋疼啊，BIOS第一条指令后，只剩16个字节了，然后跳到前面去了，估计是历史原因= =。) 然后初始化设备和PCI总线后，搜索可以启动的设备如软盘、硬盘、CD-ROM等，开始读取boot loader ...

MIT-6.828课程

Mon 30 March 2015

这个学期学习操作系统课程，教学质量实在无法令人满意。无论是课堂内容还是实验内容，都远远低于我的期望值，遂跟随MIT-6.828课程进行学习。

整个课程分为两个模块，一个是对xv6操作系统的学习，平时的homework和这个有关。另一个是自己做一个操作系统（当然一开始给了一些非常底层的代码，并且省略了一部分需要你自己完成），即所谓的Lab。

目前我只在做Lab这个模块，感觉受益匪浅。这个系列的Lab的一大优点就是自带测评，可以执行make grade来自动对自己所做的Lab评分。

一些唠叨

Tue 17 March 2015

春雷滚滚，大雨倾盆。独自撑伞，听歌，遐想。路灯昏黄，树影婆娑。朦胧诗意，我心悠然。这是我一天中最最享受的时光，超然物外，仿佛整个世界都是你的，又仿佛都不是。

QQ，班群信息，燥热之气扑面。阿里员工级别啊,听说哪哪android开发实习月薪12K啊......欲望，像一只少女的手，轻轻撩拨着一颗颗蠢蠢欲动的心。

打下几行字，又删了，想想还是算了。何必泼一盆冷水，坏了某些人兴致，招得一身嫌。

QQ，群信息，习惯性地看看一群准备进FLAG的牛人在干啥，嗯，几个人在讨论题目和相关算法。

我明白，优秀是有理由的。

前几天还忿忿地写了一篇吐槽性质的文章，看了看，还是删了。价值决定价格，有些人暂时是不会明白的。

看了看明天的计划列表，扫了眼右上角的时间，又是全新的一天。

潜下心来，闭关修炼。

你若盛开，清风自来。

解决pelican-quickstart出错问题

Fri 13 March 2015

博客框架pelican更新后，在目录下执行pelican-quickstart会报错：

ImportError: No module named html_parser

我的环境是:

OS X Yosemite (Version 10.10.2)
python 2.7.6

原因可能是与python 2.7的兼容问题,在/Library/Python/2.7/site-packages/pelican下修改readers.py即可(注意要先修改readers.py的权限):

#from six.moves.html_parser import HTMLParser
from HTMLParser import HTMLParser
