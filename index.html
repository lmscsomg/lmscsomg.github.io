解决pelican-quickstart出错问题

Fri 13 March 2015

博客框架pelican更新后，在目录下执行pelican-quickstart会报错：

ImportError: No module named html_parser

我的环境是:

OS X Yosemite (Version 10.10.2)
python 2.7.6

原因可能是与python 2.7的兼容问题,在/Library/Python/2.7/site-packages/pelican下修改readers.py即可(注意要先修改readers.py的权限):

#from six.moves.html_parser import HTMLParser
from HTMLParser import HTMLParser

使用readthedocs整理文档

Sun 26 October 2014

最近用Read The Docs整理笔记，感觉还不错，同时，sphinx主题对移动端的支持非常好。将整个文档部署到github上，我就可以随时随地查阅，非常方便。

Read The Docs支持Markdown格式，我自己用了下，感觉配套主题没有使用reStructuredText格式的配套主题效果好。reStructedText相对于Markdown而言，也更为强大，唯一的缺点就是没有类似Mou这样的配套应用（用Mou写Markdown着实方便啊）。

这里记录一下将本地文档部署到github上出现的问题：如果按sphinx主题的Readme教程，本地完全没有问题，但是在readthedocs部署时会报错，会报如下错误:

ImportError: No module named sphinx_rtd_theme

看github上的Issues列表，貌似是本地和部署到Read The Docs会有冲突(因为readthedocs默认使用这个主题？)，所以我们需要在config.py里进行判别。

import sphinx_rtd_theme
html_theme = "sphinx_rtd_theme"
html_theme_path = [sphinx_rtd_theme.get_html_theme_path()]
